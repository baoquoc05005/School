Week 4 Review

  • Overloaded methods: Overloaded methods let you reuse the same method name in a class, but with different arguments, optionally, a different return type. Remember, the rules in overloading:
    • Overloaded methods must change the argument list.
    • Optionally, overloaded methods can change the return type and the access modifier.

Here is the method that I want to overload:

public void changeSize(int size, String name, float pattern)

Filter the legal overloads:

  1. public void changeSize(int size, String name)
  2. private int changeSize(int size, float pattern)
  3. private int changeSize(int size, String name, float pattern)
  4. private void changeSize(String name, int size, float pattern)
  5. public void changeSize(float pattern, String name)

Remember, what you can’t do is change only the return type, access specifier or sequence of parameters. To overload a method, you must change the argument/parameter list.

  • Variables that you know so far…
  • Instance Variables: Instance variables are defined inside the class, but outside of any method, and are initialized only when the class is instantiated. Instance variables are the fields that belong to each unique object.

class Employee{

private String name, title;

}

Each employee instance will know its own name and title. It means, each instance has its own unique values for those two fields.

  1. Local Variables: A local variable is a variable declared within a method. That means the variable is not just initialized within the method, but also declared within the method.

public void logIn(){

int count = 10;

}

public void doSomething(int i){

count = i; //won't compile, can't access count outside method logIn

}

  1. Static Variables: A static modifier is used to create variables and methods that will exist independently of any instances created for the class. There will be only one copy of a static member regardless of the number of instances of that class. The bottom line is that all instances of a given class share the same value for any given static variable.
  • The common practice is to name a parameter with the same name as the instance variable to which the parameter will be assigned.

The following code is trying to set an instance variable’s value using a parameter:

class MyClass{

private int size;

public void setSize(int size){

size = size; // which size equal to which size??

}

}

Use keyword this to resolve the naming collision. The keyword this always refers to the object currently running.

class MyClass{

private int size;

public void setSize(int size){

this.size = size; // this.size means the current object's instance variable, size. // the size on the right is the parameter.

}

}

  • static method can’t access a nonstatic (instance) variable because a static method does not know anything about an instance.
  • static method can’t directly invoke a nonstatic method. To summarize, static = class, nonstatic = instance. Making the method called by the JVM (main()), a static method means the JVM does not have to create an instance of your class just to start running code.
  • access a static method or static variable , use the dot operator on the type name, as opposed to using it on a reference to an instance.
